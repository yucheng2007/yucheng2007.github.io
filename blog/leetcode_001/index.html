LeetCode #1 Two Sum

By JJ.Chang 2023-10-13 Leet code
原文題目：

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target.

You may assume that each input would have exactly one solution, and you may not use the same element twice.

You can return the answer in any order.

題目解釋與目的：

給定一個整數數組 nums 和一個整數 target，要求找出數組中兩個數字的索引，使它們的和等於目標值 target。假設每個輸入只有一個解，而且不能使用相同的元素兩次。需要返回這兩個數字的索引，按非降序排列。

這個問題的目的是在給定數組中找到兩個數字，使它們的和等於目標值，並返回這兩個數字的索引。

限制條件：
  • 陣列中只有一組解
  • 不能使用同一個元素兩次

接下來我們會分別用C#和Python來解答，並提供每段程式碼的詳細註解。

C# 解答：
解釋：
  1. 使用 Dictionary 來存儲數字及其對應的索引。
  2. 遍歷數組，對於每個數字，計算與目標值的補數。
  3. 如果補數存在於字典中，返回補數的索引以及當前數字的索引，即為答案。
  4. 如果不在字典中，將當前數字及其索引存入字典。
  5. 如果沒有解，拋出異常。
Python 解答：
解釋：
  1. 使用字典來存儲數字及其對應的索引。
  2. 遍歷數組，對於每個數字，計算與目標值的補數。
  3. 如果補數存在於字典中，返回補數的索引以及當前數字的索引，即為答案。
  4. 如果不在字典中，將當前數字及其索引存入字典。
  5. 如果沒有解，拋出異常。

接下來我們來總結一下這題的目的和解題技巧。

解題總結：

這個問題的目的是要在給定的數組中找到兩個數字，使它們的和等於目標值。使用哈希表可以將查找時間從O(n)降低到O(1)，提高了解題效率。該題展示了哈希表的優越性，特別適用於需要快速查找的情境。

補充說明：

哈希表（Hash Table）是一種常用的資料結構，用於儲存鍵值對（key-value pairs）。它通過將鍵（key）進行哈希函數運算，將結果映射到表中的一個位置，再在該位置進行資料的儲存或查詢。這樣可以在平均情況下以常量時間複雜度（O(1)）進行資料的查詢、插入和刪除操作。

哈希表的主要組成部分包括：

  1. 哈希函數（Hash Function）： 將鍵轉換為特定範圍的索引，通常是一個整數。這個轉換過程稱為哈希化。一個好的哈希函數可以最大程度地避免碰撞，即不同的鍵經過哈希函數映射後得到相同的索引。

  2. 哈希表（Hash Table）： 是由多個哈希桶（buckets）組成的資料結構，每個哈希桶存儲多個鍵值對。哈希桶可以是陣列、連結列表或其他資料結構。

碰撞解決方法（Collision Resolution）： 因為哈希函數可能產生相同的索引（碰撞），所以需要一種方法來處理碰撞。常用的碰撞解決方法有鏈接法（使用連結列表解決碰撞）和開放地址法（在發生碰撞時尋找下一個空位置）。

哈希表的優點是能夠快速進行查詢、插入和刪除操作，具有良好的平均時間複雜度。然而，在某些情況下可能會有較差的最壞情況時間複雜度，特別是當哈希函數導致大量碰撞時。選擇適合的哈希函數和碰撞解決方法對於保證哈希表效率至關重要。
